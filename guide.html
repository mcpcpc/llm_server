LLM Server -- User Guide

Self-hosted AI coding assistant · Powered by Qwen3.6-35B-A3B · GE HealthCare Engineering

Overview

This server provides access to a high-performance, locally-hosted large language model optimised for software engineering tasks. All inference runs on-premises on an NVIDIA DGX Spark -- your code and prompts never leave the network.

🔒

Private by default

On-premises inference. No data sent to third-party APIs.

⚡

Fast & capable

35B MoE model with speculative decoding. Optimised for coding.

🔌

OpenAI-compatible

Works with any tool that supports a custom OpenAI API endpoint.

📊

Usage tracked

Per-user API keys with token usage and budget visibility.

continue.dev Recommended

continue.dev is the recommended client for inline code completion and chat. Install the VS Code or JetBrains extension, then add the following to your ~/.continue/config.yaml:

Chat Model
Autocomplete

Add under the models section of your config for chat and inline assist:

# ~/.continue/config.yaml
models:
  - name: LLM Server (Qwen3.6)
    provider: openai
    model: qwen3.6-35b-a3b
    apiBase: http://SERVER_IP:4000/v1
    apiKey: sk-your-key-here

Add under the tabAutocompleteModel section for fast inline completions:

# ~/.continue/config.yaml
tabAutocompleteModel:
  name: LLM Server (Autocomplete)
  provider: openai
  model: qwen3.6-35b-a3b
  apiBase: http://SERVER_IP:4000/v1
  apiKey: sk-your-key-here
Tip: In VS Code, open the continue.dev sidebar, click the model selector at the bottom, and choose LLM Server (Qwen3.6) to activate it.

cURL / Direct API Access

You can call the API directly from the terminal or any HTTP client. The endpoint is fully OpenAI-compatible.

Basic chat completion

curl http://SERVER_IP:4000/v1/chat/completions \
  -H "Authorization: Bearer sk-your-key-here" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen3.6-35b-a3b",
    "messages": [
      {"role": "user", "content": "Write a Python function to parse JSON"}
    ],
    "stream": false
  }'

Streaming response

curl http://SERVER_IP:4000/v1/chat/completions \
  -H "Authorization: Bearer sk-your-key-here" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen3.6-35b-a3b",
    "messages": [
      {"role": "user", "content": "Explain async/await in Python"}
    ],
    "stream": true
  }'

Python (openai SDK)

from openai import OpenAI

client = OpenAI(
    base_url="http://SERVER_IP:4000/v1",
    api_key="sk-your-key-here"
)

response = client.chat.completions.create(
    model="qwen3.6-35b-a3b",
    messages=[
        {"role": "user", "content": "Review this code for bugs"}
    ]
)
print(response.choices[0].message.content)

Usage Tips

🧠 Thinking mode

Qwen3.6 supports chain-of-thought reasoning. For complex problems, ask it to "think step by step" -- the model will reason through the problem before responding, improving accuracy on hard tasks.

📄 Large context

With a 262K token context window, you can paste entire files or large codebases. This is particularly useful for code review, refactoring, or understanding unfamiliar projects.

🔧 Tool calling

The server has tool calling enabled. Agentic tools like Pi Agent can use this for multi-step tasks -- planning, writing code, running tests, and iterating -- without manual intervention.

⚡ Prefix caching

If you send the same system prompt repeatedly (e.g. a coding persona or project context), the server caches it automatically. Subsequent requests with the same prefix will be faster.

🔑 Key security

Never hardcode your API key in source files. Use environment variables (OPENAI_API_KEY) or a secrets manager. If you believe your key is compromised, contact the administrator immediately.
